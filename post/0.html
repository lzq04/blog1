决策规划

  • 决策规划算法就是普通L2到什么都能做的L2一个重要模块

感知: 人的眼睛，耳朵

控制： 人的小脑，双脚

决策规划：人的大脑

功能越往上做，越复杂，越丰富，决策规划算法越重要，越难

在L4中决策规划模块可以说是最重要的，也是最复杂，最难做的模块

决策规划算法就是普通L2到什么都能做的L2的一个重要模块

整个决策规划模块一分为三

  1. 导航规划算法（业内叫全局路径规划算法），此算法将计算出地图上A到B的最优路径，此算法与机器人导航，手机导航算法基本一致，长度在几公里到几百公里不等，该算法是整个规划模块中最为成熟的算法

    特点：导航规划算法规格一个粗略的，大范围的路径，但是此路径不会考虑如何避障，也不会考虑车辆运动学的约束，一般规划的路径是不不规则的折线，导航算法一般只需要执行一次，只有遇到大范围的拥堵，施工，偏航等情况才会再次执行。

  2. 行为规划算法，又叫决策算法，决定车辆行驶意图的算法，对于静态障碍物，到底是左绕，还是右绕，对于动态障碍物，到底是该减速避让，还是加速超车？决策算法决定了车辆的意图，这也是规划算法中最难做的部分
    特点：决策算法给一个车辆的行驶意图，会指导车辆该避让还是该超车，该左转还是右转，但是决策不会给具体的运动建议，例如往左转多少度，车辆加速多少。实际环境瞬息万变，因此决策算法需要较高的执行频率，一般为10Hz，决策算法需要有一定的稳定性，不允许在周围环境稳定的情况下出现“朝令夕改”的现象。

  3. 运动规划算法：根据决策给出的行为意图在相关的时空搜索出（或优化出）一条具有详细路径速度信息，并且满足各个约束条件的轨迹，并将此轨迹发给控制模块。此轨迹一般在几米到几十米不等
    特点：运动规划生成的轨迹是决策规划模块的最终输出，具有详细的路径速度的信息，执行频率与决策相同，为10Hz，同样运动规划也要求具有一定的稳定性。

EM Planner是Apllo的经典规划算法，擅长处理复杂环境下的决策规划问题，也是Apollo默认的决策规划算法。

截至到2021年，Apollo已经迭代到6.0，EM Planner 在 Apollo1.5加入，目前6.0的EM Planner 与 1.5相比已经发生了一些变化，现在6.0的EM Planner 换了个名字叫Onlane Planning

1.数学基础

第一节 细说五次多项式

五次多项式是规划论文的常客，本节将详细解释五次多项式的特殊性

tip： 本节难度较大，但并不重要，听不懂记主结论即可。

车辆运动规划中，一个非常重要的指标就是舒适性，在物理中，衡量舒适性的物理量为跃度，英文为Jerk

Jerk的定义为：

​ Jerk的定义是
$$
Jerk = \frac{\text{d} a}{\text{d} t} \
a为加速度，t为时间。
$$
即Jerk是加速度的导数，Jerk的绝对值越小意味着a的变化越平缓，a的变化越平缓，也就意味着越舒适。

二次规划

自动驾驶规划目标：算出一条满足约束的最优轨道

什么是最优？

指标

  1. 平滑性
  2. 舒适性
  3. 尽可能短，耗时少

约束

  1. 轨道连续性
  2. 无碰撞
  3. 交规
  4. 车辆动力学

  1. 为什么要讲凸优化？
    凸优化是最简单的，非线性的规划方法，也是目前研究的比较明白，比较彻底的规划方法，凸优化是很多复杂非线性规划方法的基石。
  2. 凸优化和自动驾驶规划算法有什么联系？

静态 + 动态 避障约束空间都不是凸的。

第二章

1 . 决策规划仿真平台搭建

  • 需要用到的软件：

    1. vs2019
    2. carsim2019.1
    3. matlabr2020a
    4. prescan8.5.0

  • 搭建中可能出现的问题：

    1. matlab异常，需要重新安装，
    2. carsim在打开之前一定要记得重启
    3. 修改prescan的.dat文件为当前年份
    4. 在进行仿真的过程中碰到问题，一定要线看评论区的6个问题

  • Q1:prescan安装了，但是找不到Demo_Carsim3D
    A1:这个文件夹是我自己建立的不是prescan自带的，所以你们安装自然是没有的(这是我的错误，向大家道歉)，所以以后视频里所有的操作都应该等价转化为视频07:27倒数第四个文件夹Demo_PrescanCarsim3D，包括Carsim输入输出模块的粘贴也要用Demo_PrescanCarsim3D下的模型

    Q2:Carsim2019.0与prescan关联报错there were unbalanced () from the expression
    A2:(此问题由@石榴_SHILIU解决)可以尝试在Export: Outputs右下角Optional equations添加
    DEFINE_OUTPUT mod1_L1 = 0; - ; mod1_L1
    DEFINE_OUTPUT mod2_L1 = 0; - ; mod2_L1
    EQ_OUT MOD1_L1 = (ABS(FY_L1)/FZ_REF(1,1))*MUY_L1;
    EQ_OUT MOD2_L1 = SIN(MIN(ABS(ALPHL_L1), 15/DR))*ABS(VXCENL1);

    Q3: prescan的动力学模型中，点进去里面的Carsim S-Funtion 图标显示Solver_SF not found
    A3:请确定matlab是否添加了路径X:\Program Files (x86)\CarSim2019.1_Prog\Programs\solvers(X为你carsim安装的盘符，默认是C盘)，这个路径就是Solver_SF的路径，如果不知道怎么在matlab添加路径请搜索“matlab如何设置路径”，如果做了这一切还是不行，请按照视频的方式，看改名字把Carsim S-function2改成Carsim S-function，或者把Carsim S-function改成Carsim S-function2，如果还是不行，双击Carsim S-function这个s函数，在source block中，改Solver_SF/Carsim S-function的名字Q4:carsim破解报错The hostid of this system does not match the hostid
    A4:请打开X:\Program Files (x86)\CarSim2019.1_Prog\Programs(X默认为C盘),运行HostID.exe，查看你电脑的HostID，以BIA开头，一共有两个，选择一个即可，在用记事本打开破解文件MSCLIC_SSQ.dat 查找此文件的HOST ID= BIAXXXXXX，把BIAXXXXXX替换成你自己电脑的HostID，因为要替换的HostID不止一处，所以要选择全部替换，即可解决。

    Q5:一切准备就绪之后，模型运行报错 ”函数或变量vs_state无法识别“
    A5:在工作区加上这个变量即可，具体操作为在matlab的命令行输入vs_state=-1,回车，然后运行，但是他还是会报错说”函数或变量StopMode无法识别“，然后再在命令行输入StopMode=-1，即可运行，vs_state和StopMode参数的具体含义可以搜索carsim的帮助文档，在这个系列这两个参数我们用不着

    Q6:顺利解决所有问题，但是没跑出来视频中的曲线,而且仿佛车不受控制
    A6:请检查carsim的纵向和横向控制是否关掉，carsim可能自带一些初速度和转向的设置，在仿真时请把他们都关掉

2. 参考线模块

参考线是解决导航路径过长，不平滑的问题

过长的路径不利于坐标转换（找匹配点，计算S）

过长的路径，障碍物投影可能不唯一

不平滑

每个规划周期内，找到车在导航路径上的投影点，以投影点为坐标原点，往后取30米，往前取150米范围内的点做平滑，平滑后的点的集合称为参考线。
